# Simplified Technical English (ASD-STE100)

Use STE100 principles for technical content in this project: rules, checklists, component labels, error text, and lesson instructions.

## Goals

- Short sentences
- Active voice
- Controlled vocabulary
- One idea per sentence
- No ambiguity

## Do

- Write: `Use a type scale.`
- Write: `Make the title the strongest text.`
- Write: `Keep body text at 15px or 16px.`
- Prefer concrete verbs: use, set, keep, show, hide, align, group.
- Prefer numbers and exact tokens over vague words.

## Do not

- Long nested clauses
- Passive voice when active is clear
- Marketing fluff in rule lists
- Synonyms for the same technical term in one lesson
- Vague words: optimize, leverage, seamless, robust (in rules)

## Where it applies

| Content | STE100 |
|---------|--------|
| Lesson rules / checklists | Required |
| Demo labels (Good / Bad / token names) | Required |
| Error and empty states | Preferred |
| Home marketing intro | Light touch (still clear and sober) |
| Code comments | Preferred for intent comments |

## Example

Bad:

> Hierarchy is essentially about ensuring that users can seamlessly understand what matters most as they navigate complex interfaces.

Good:

> Type hierarchy shows what matters first. Users scan titles before body text.
